Módulos

¿Qué es un Módulo?

Un módulo en Angular es un contenedor que agrupa componentes, directivas, pipes y otros módulos. Permite organizar la aplicación en partes reutilizables y gestionar las dependencias.

Cada módulo se define en un archivo con extensión .module.ts y normalmente tiene la siguiente estructura:

  • declarations: Lista de componentes, directivas y pipes que pertenecen al módulo.
  • imports: Otros módulos que se necesitan para este módulo.
  • exports: Componentes, directivas o pipes que estarán disponibles fuera del módulo.
  • providers: Servicios que proporciona este módulo.
  • bootstrap: Componente raíz (solo en AppModule).

Ejemplo de módulo

Supongamos un módulo llamado AppModule:

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule {}

- declarations incluye todos los componentes que pertenecen al módulo.
- imports indica otros módulos que necesitamos (como BrowserModule para aplicaciones web).
- providers es donde se agregan los servicios que se usarán en el módulo.
- bootstrap indica cuál es el componente inicial al arrancar la aplicación.

Usar un módulo

Una vez declarado un módulo, podemos importarlo en otros módulos para reutilizar componentes, directivas o pipes. 

import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { SharedComponent } from './shared.component';

@NgModule({
  declarations: [SharedComponent],
  imports: [CommonModule],
  exports: [SharedComponent]
})
export class SharedModule {}

En este ejemplo, SharedModule exporta SharedComponent, de manera que cualquier otro módulo que importe SharedModule pueda usarlo.
